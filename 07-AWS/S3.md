# Amazon S3 (Simple Storage Service)

## Introduction

Amazon S3 (Simple Storage Service) is AWS's object storage service.

It is designed to store and retrieve any amount of data from anywhere on the internet.

S3 is one of the most widely used AWS services.

---

# What is S3?

S3 is an object storage service.

Data is stored as:

```text
Objects
```

inside:

```text
Buckets
```

---

# S3 Architecture

# Important Terminology

## Bucket

A container that stores objects.

Example:

```text

```

---

## Object

A file stored inside S3.

Example:

```text

```

---

## Key

Unique name of an object.

Example:

```text

```

# Interview Question

## What is Amazon S3?

### Answer

Amazon S3 (Simple Storage Service) is a highly scalable object storage service that allows users to store and retrieve data from anywhere on the internet.

---

# Interview Question

## What is the Difference Between Object Storage and Block Storage?

### Answer

S3 uses object storage where data is stored as objects inside buckets. EBS uses block storage where data is stored as blocks attached to EC2 instances.

---

# Interview Question

## What is Versioning in S3?

### Answer

Versioning allows multiple versions of an object to be stored in a bucket, protecting against accidental deletion and unintended modifications.

---

# Interview Question

## What is a Lifecycle Policy?

### Answer

A Lifecycle Policy automatically transitions objects between storage classes or deletes them after a specified period to optimize storage costs.

---

# Interview Question

## What is Cross-Region Replication?

### Answer

Cross-Region Replication automatically copies objects from one S3 bucket to another bucket in a different AWS Region for disaster recovery and high availability.

---

# Interview Question

## Which Storage Class Would You Use for Backups?

### Answer

For backups that are rarely accessed, I would use Glacier or Glacier Deep Archive because they provide very low storage costs.

---
